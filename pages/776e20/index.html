Java反射特性摘要

知识点来自P牛的《Java安全摘要》，这篇文章是其反射部分的一些摘要

# 反射(一)

反射中一些极为重要的方法：

  • 获取类的方法：forName
  • 实例化类对象的方法：newInstance
  • 获取函数的方法：getMethod
  • 执行函数的方法：invoke

反射机制中的Class.forname()中有两种形式：

Class.forname(className)
// 等于
Class.forname(className, true, currentLoader)

# 反射(二)

$的作用是查找内部类，比如在C1类中存在内部类C2,通过Class.forname("C1$C2")就可以加载该内部类

Class.newInstance()会调用这个了类的无参构造函数，但有的时候并不会成功，主要原因可能是：

  • 使用的类没有无参构造函数
  • 该类的构造函数是私有的

比如Java.lang.Runtime类，如果直接new Runtime()会产生错误

笔记

这种把类的构造方法权限设置为“私有”，这种设计模式称为单例模式

所以Runtime类就是“单例模式”，所以我们只能通过Runtime.getRuntime()来获取这个类的对象

// 由于单例模式，执行失败
Class<?> runtime = Class.forName("java.lang.Runtime");
runtime.getMethod("exec",String.class).invoke(runtime.newInstance(), "calc");

// 执行成功
Class<?> runtime = Class.forName("java.lang.Runtime");
runtime.getMethod("exec", String.class).invoke(runtime.getMethod("getRuntime").invoke(runtime), "calc");

下面执行成功的那一段代码的逻辑就是：

  • 通过runtime.getMethod("gerRuntime").invoke(runtime)获取到Runtime类的对象
  • 把这个对象作为runtime.getMethod("exec", String.class).invoke(···)的第一个参数
  • calc作为第二个参数，执行系统命令

关于invoke：

  • 如果这个方法是一个普通方法，那么第一个参数是对象
  • 如果这个方法是一个静态方法，那么第一个参数是类

笔记

如果正常调用一个方法的形式是Instance.method(para)，那么通过invoke调用的形式就是method.invoke(Instance, para)

总结

这篇文章解决了一个问题：

  • 如何通过反射创建出单例模式设计的类

但是仍然有两个问题没有解决：

  • 如果一个类没有无参构造方法，也不是单例模式，能否通过反射来获取这个类的对象？
  • 如果一个类的无参构造方法权限是私有，那么是否可以通过某种手段执行呢？

# 反射(三)

# 遗留的第一个问题

getConstructor方法可以获取类的构造函数，它接受的参数是构造函数列表类型，因为构造方法也是支持重载的，所以必须通过参数列表类型来确定唯一的构造方法。当通过这个方法获取到构造方法之后可以通过newInstance来执行，其中的参数就是获取到的构造方法的参数。

比如，另一个常见的执行命令的类ProcessBuilder，它没有无参构造方法，也并不是像Runtime类那样的"单例模式"，这时候就无法通过ProcessBuilder.newInstance()来获取这个类的对象。这时候就需要通过getConstructor方法来获取构造函数，利用获取到的构造函数来执行newInstance方法，然后才可以创建类型的对象。

ProcessBuilder类有两个构造方法：

public ProcessBuilder(List<String> command)

public ProcessBuilder(String... command)
1
2
3

反射执行系统命令

Class<?> proClass = Class.forName("java.lang.ProcessBuilder");
proClass.getConstructor(List.class).newInstance(Arrays.asList("calc"));
// java.lang.ProcessBuilder@2133c8f8
proClass.getMethod("start").invoke((proClass.getConstructor(List.class).newInstance(Arrays.asList("calc"))));
1
2
3
4

如果要反射另一个构造方法public ProcessBuilder(String... command)的话，需要先了解一下Java中的可变长参数

对于可变长参数，Java会将其编译成一个数组，所以下面这两种写法是等价的

public void hello(String[] names) {}
public void hello(String... names) {}
1
2

P牛：那么对于反射来说，如果要获取的目标函数里包含可变长参数，其实我们认为它是数组就行了。

所以可以通过数组的形式来反射第二个构造方法

proClass.getConstructor(String[].class)
1
Class<?> proClass = Class.forName("java.lang.ProcessBuilder");
proClass.getMethod("start").invoke((proClass.getConstructor(String[].class).newInstance(new String[][]{{"calc"}})));
1
2

特别值得注意的是，在调用newInstance的时候，这个函数本身接受的是一个可变长参数，同时传给ProcessBuilder的也是一个可变长参数，所以两者会进行叠加，也就是一个二维数组。

# 遗留的第二个问题

getDeclaredMethod方法与getMethod的区别就是（getDeclaredConstructor方法类似）：

  • getMethod方法获取的是当前类中所有的公共方法，包括从父类中继承的方法
  • getDeclaredMethod方法获取的是在当前类中声明的方法，包括私有方法，但是不会获取父类中的方法

Runtime中的构造方法是私有的，但是现在我们可以直接通过getDeclaredMethod方法获取到这个私有的构造方法从而获取Runtime类的对象

当我们获取到私有方法之后，需要设置setAccessible(true)来修改私有方法的访问权限
